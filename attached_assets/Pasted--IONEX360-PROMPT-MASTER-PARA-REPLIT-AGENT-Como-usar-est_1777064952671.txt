## INÍCIO DO PROMPT — cole a partir daqui no Replit Agent

Você vai construir o **IONEX360 Command Center** — uma plataforma SaaS multi-tenant para consultorias de governança que atendem clínicas médicas. O sistema é operado pela **Blu Sollutions** (consultoria) e usado por suas clínicas-cliente.

Primeiro cliente real: **Instituto de Cardiologia de Sorriso (ICS)** — contrato assinado, R$ 10.000/mês nos primeiros 3 meses + R$ 5.000/mês de recorrência.

## 1. STACK TÉCNICO (OBRIGATÓRIO)

- **Styling:** Tailwind CSS + shadcn/ui (dark mode default)

## 3. ROLES E PERMISSÕES

4 roles, com hierarquia:

| Role | Pode ver | Pode editar |
|---|---|---|
| **super_admin** | Tudo, incluindo gestão de todas as clínicas | Tudo |
| **consultor** | Clínica ativa: todos os módulos | Tudo exceto Admin de clínicas |
| **gestor** | Clínica ativa: todos menos Pilares/Perguntas | Delegar sub-tarefas, responder, enviar evidências |
| **equipe** | Command Center, Diagnóstico (só suas perguntas), Plano de Ação (suas tarefas), Evidências | Responder, enviar evidências, atualizar status de tarefas suas |

O **super_admin** pertence à **Blu Sollutions** (tenant da consultoria).
**Consultor, gestor, equipe** pertencem a **uma clínica específica** (tenant da clínica).

RLS do Supabase deve garantir isolamento por `clinic_id`.

---

## 4. ARQUITETURA MULTI-TENANT

- Tabela `clinics` é o **tenant principal**. Cada linha = uma clínica cliente.
- Todas as tabelas operacionais (delegacoes, riscos, acoes, etc.) têm `clinic_id` com FK + RLS.
- Tabela `organization_users` liga usuários a clínicas com role.
- Um usuário pode estar em MÚLTIPLAS clínicas (ex: Rafaela Calgaro é super_admin e também consultora em várias clínicas).
- Contexto de clínica ativa fica em cookie/localStorage. Switcher no topbar.

### 5.1. Super Admin — Gestão de Clínicas (CRM)

**Edição (`/admin/clinicas/[id]`):** 5 abas

#### Aba 1 — Cadastro (com BrasilAPI)

- Input CNPJ com botão "Buscar na Receita"
- Ao clicar: `fetch('https://brasilapi.com.br/api/cnpj/v1/' + cnpjLimpo)`
- Preencher automaticamente: razão social, nome fantasia, endereço completo, CEP, município, UF, CNAE, data abertura, capital social, situação cadastral, **QSA (quadro societário)**
- Mostrar indicador de loading → success/error
- Campos editáveis após retorno
- Seção "Responsável principal" (contato): nome, cargo, email, WhatsApp

#### Aba 2 — Financeiro & Contrato

- Upload Proposta (PDF) + Upload Contrato (PDF)
- Status do contrato: rascunho / enviado / assinado
- **Botão "Enviar via Autentique"** → Edge Function chama API Autentique (ver seção 7.2)
- Valor implantação (R$ 10.000 para ICS)
- Valor recorrente (R$ 5.000)
- Data início recorrência (mês 4)
- Forma de pagamento: Pix, Boleto, Cartão, TED (radio)
- Dia de vencimento
- Índice de reajuste (default IGPM/FGV)

#### Aba 3 — Status (ciclo de vida)

Estados possíveis: `prospect → proposta → contrato → trial → ativa → suspensa → desativada`

Ações disponíveis:
- **Suspender** (apenas se ativa): bloqueia acesso, mantém dados. Requer motivo.
- **Reativar** (apenas se suspensa): restaura acesso.
- **Desativar** (qualquer estado): encerra contrato. Dados disponíveis por export por 30 dias (Cláusula 14 do contrato), depois acesso é revogado. Requer motivo. Confirmação dupla.

Cada mudança grava linha em `clinic_status_history` com: de, para, data, autor, motivo.

Timeline vertical do histórico visível.

Alerta destacado explicando a cláusula 14 de retenção de dados.

#### Aba 4 — Usuários

Lista de usuários da clínica com: nome, email, role, último acesso, status (ativo/convite pendente).
- Botão "Convidar usuário" (envia magic link via Supabase Auth)
- Revogar acesso
- Reenviar convite

#### Aba 5 — Atividade

Timeline vertical com eventos importantes:
- Clínica cadastrada

## 6. PILARES E PERGUNTAS (SEED)

Use o CSV `diagnostico_pilares_questoes.csv` (150 linhas) como fonte. Script para carregar:

```python
# scripts/seed_perguntas.py
import csv, os
from supabase import create_client

sb = create_client(os.environ['SUPABASE_URL'], os.environ['SUPABASE_SERVICE_ROLE_KEY'])

with open('diagnostico_pilares_questoes.csv') as f:
    reader = csv.DictReader(f)
    for row in reader:
        sb.table('perguntas').insert({
            'pilar_slug': row['pilar_slug'],
            'texto': row['texto'],
            'tipo': row['tipo'],   # sim_nao, escala_1_5, texto_livre, numerico
            'peso': int(row['peso']),
            'ordem': int(row['ordem']),
        }).execute()
```

Se o CSV não estiver disponível, **gere uma versão mínima com 5 perguntas por pilar** (40 total) como placeholder e marque TODO para substituir.

## 7. INTEGRAÇÕES EXTERNAS

### 7.1. BrasilAPI (CNPJ)

- **URL:** `https://brasilapi.com.br/api/cnpj/v1/{cnpj}` (sem token, CORS-friendly)
- **Chamada:** client-side direto via fetch
- **Resposta principal:** `razao_social`, `nome_fantasia`, `logradouro`, `numero`, `bairro`, `municipio`, `uf`, `cep`, `cnae_fiscal`, `cnae_fiscal_descricao`, `data_inicio_atividade`, `capital_social`, `descricao_situacao_cadastral`, `qsa[]` (array de sócios)
- **Sem quota** — 100% gratuito

### 7.2. Autentique (assinatura digital)

- **URL:** `https://api.autentique.com.br/v2/graphql`
- **Auth:** Header `Authorization: Bearer {AUTENTIQUE_API_TOKEN}`
- **Edge Function Next.js:** `/api/autentique/create-document`
- **Fluxo:**
  1. Upload do PDF para Supabase Storage
  2. Chama Autentique via GraphQL mutation `createDocument`
  3. Recebe `document.id` + `signatures[].link`
  4. Envia link por email (Resend) para cada signatário
  5. Webhook `/api/autentique/webhook` recebe updates de status
  6. Atualiza `assinaturas.status` no banco em tempo real
- **Status possíveis:** aguardando, visualizado, assinado, concluido, expirado, recusado
- **Plano:** 5 assinaturas grátis/mês, R$ 0,30/doc depois

Esquema GraphQL mínimo para criar documento:
```graphql
mutation {
  createDocument(
    document: { name: "Contrato ICS" }
    signers: [
      { email: "edgar@ics.com.br", action: SIGN, positions: [...] }
      { email: "rafaela@blu.com.br", action: SIGN }
    ]
    file: $file
  ) {
    id
    signatures { public_id link { short_link } email }
  }
}
```

### 7.3. Claude API (insights IA)

- **URL:** `https://api.anthropic.com/v1/messages`
- **Model:** `claude-opus-4-5` (ou o mais recente disponível)
- **Edge Function:** `/api/ai/analyze-diagnostico`
- **Input:** JSON com scores dos 8 pilares + respostas críticas
- **Output estruturado:** 
  ```json
  {
    "pontos_fortes": ["..."],
    "pontos_criticos": ["..."],
    "acoes_sugeridas": [{"titulo":"...","pilar":"...","prio":"alta"}]
  }
  ```
- Mostrar indicador "Gerado pela IA · Claude" com disclaimer

### 7.4. Email (Resend)

- Envio transacional (convites, notificações de delegação, faturas, alertas de vencimento)
- Templates em React Email (`@react-email/components`)
- Domain: `ionex360.com.br` (configurar SPF/DKIM depois)

### 7.5. WhatsApp (Meta Cloud API)

- Notificações de delegação e aprovações
- Templates pré-aprovados (iniciar com 2-3 templates)
- Fallback: se falhar, só envia email
