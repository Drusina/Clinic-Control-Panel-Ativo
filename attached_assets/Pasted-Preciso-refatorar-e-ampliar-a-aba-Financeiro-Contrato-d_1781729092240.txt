FLUXO GERAL

O fluxo deve funcionar assim:

1. Clínica já cadastrada no sistema.
2. Usuário acessa a aba “Financeiro & Contrato”.
3. Sistema exibe os dados comerciais editáveis.
4. Usuário preenche ou altera as condições comerciais.
5. Usuário salva as condições comerciais.
6. Usuário clica em “Gerar Proposta”.
7. Sistema gera PDF da proposta comercial usando:

   * dados da clínica já cadastrada;
   * condições comerciais preenchidas;
   * modelo padrão da proposta CLINIONEX360.
8. Usuário visualiza a proposta.
9. Usuário envia a proposta via Autentique.
10. Sistema acompanha status da proposta.
11. Após proposta assinada/aceita, sistema libera botão “Gerar Contrato”.
12. Usuário pode revisar condições comerciais antes de gerar o contrato.
13. Sistema gera PDF do contrato usando:

* dados da clínica;
* condições comerciais finais;
* modelo padrão do contrato CLINIONEX360.

14. Usuário visualiza o contrato.
15. Usuário envia contrato via Autentique.
16. Sistema acompanha status do contrato.
17. Após contrato assinado, sistema muda status da clínica para:
    “Contrato assinado / Kick-off pendente”.
18. Após contrato assinado, liberar:

* criação de faturas;
* avanço para etapa de Kick-off;
* botão “Agendar Kick-off” ou “Ir para Kick-off”.

VALIDAÇÕES IMPORTANTES

Antes de gerar proposta:

* verificar se existem condições comerciais salvas;
* verificar se a clínica possui razão social;
* verificar se a clínica possui CNPJ;
* verificar se existe e-mail do representante ou signatário;
* verificar se existe valor de implantação;
* verificar se existe valor mensal;
* verificar se existe prazo contratual;
* verificar se existe validade da proposta.

Antes de enviar proposta:

* verificar se PDF foi gerado;
* verificar se há signatário;
* verificar se documento ainda está válido;
* verificar se token Autentique está configurado.

Antes de gerar contrato:

* verificar se proposta está assinada/aceita;
* verificar se condições comerciais não foram alteradas sem confirmação;
* permitir revisão final.

Antes de enviar contrato:

* verificar se PDF foi gerado;
* verificar se há signatários obrigatórios;
* verificar se token Autentique está configurado.

Após contrato assinado:

* atualizar status da clínica;
* liberar kick-off;
* liberar faturas;
* registrar log;
* exibir alerta:
  “Contrato assinado com sucesso. A etapa de Kick-off foi liberada.”

MENSAGENS DE ALERTA

Quando tentar gerar contrato sem proposta aceita:
“Contrato bloqueado. É necessário que a proposta esteja assinada/aceita antes da geração do contrato.”

Quando alterar condição comercial após proposta gerada:
“As condições comerciais foram alteradas após a geração da proposta. Gere uma nova versão antes de enviar ou confirme que deseja manter a versão anterior.”

Quando proposta expirar:
“A proposta expirou. Gere uma nova versão antes de reenviar ao cliente.”

Quando contrato assinado:
“Contrato assinado. A clínica está liberada para a etapa de Kick-off.”

Quando Autentique falhar:
“Não foi possível enviar o documento ao Autentique. Verifique os dados dos signatários, o token da API e tente novamente.”
